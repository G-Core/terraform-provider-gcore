# Router Route Deletion Bug Reproduction

**JIRA Ticket:** GCLOUD2-21144
**Issue:** Routes are not deleted when removed from Terraform configuration

## Summary

This directory contains a complete reproduction case for the router route deletion bug reported by Kirill Tsaregorodtsev on November 3, 2025.

### The Bug

When a route is removed from a `gcore_cloud_network_router` resource configuration and `terraform apply` is run:
- **Expected:** Route should be deleted from the router
- **Actual:** Terraform reports "Apply complete" but the route remains on the router

## Files

1. **`reproduce_bug.sh`** - Automated script that demonstrates the bug
2. **`INVESTIGATION_SUMMARY.md`** - Detailed technical analysis of the root cause
3. **`main.tf`** - Terraform configuration (generated by the script)

## How to Run

### Prerequisites
- Valid GCore API credentials in `.env` file
- Terraform provider built from the `bugfix/terraform-lbpool` branch

### Build Provider
```bash
cd /Users/user/repos/gcore-terraform
go build -o terraform-provider-gcore
```

### Run Reproduction Script
```bash
cd test-router-route-deletion
./reproduce_bug.sh
```

The script will:
1. Create a router with an interface (no routes)
2. Add a route: `10.0.3.0/24` → `192.168.0.1`
3. Remove the route from the configuration
4. Check if the route was actually deleted (**THIS IS WHERE THE BUG SHOWS**)
5. Cleanup all resources

## Root Cause Analysis

See `INVESTIGATION_SUMMARY.md` for full details.

**TL;DR:** The `Update` function in `/internal/services/cloud_network_router/resource.go` handles routes via PATCH requests, but the GCore API likely doesn't interpret `routes: []` as "delete all routes".

Routes need special handling similar to how interfaces use `attach`/`detach` endpoints:
- Interfaces: Use `POST /v1/routers/{...}/attach` and `POST /v1/routers/{...}/detach`
- Routes: Should likely use a dedicated delete endpoint or special API handling

## Key Code Locations

- **Router Update Logic:** `/internal/services/cloud_network_router/resource.go:109-298`
- **Interface Attach/Detach:** `/internal/services/cloud_network_router/resource.go:128-222`
- **Routes PATCH:** `/internal/services/cloud_network_router/resource.go:226-267`
- **Marshal for Update:** `/internal/services/cloud_network_router/model.go:34-40`

## Recommended Fix

Check the GCore API documentation to determine if routes need:
1. A dedicated endpoint for add/remove operations (like interfaces)
2. Special handling of `routes: []` vs `routes: null`
3. Individual route deletion via DELETE endpoint

If routes require dedicated endpoints, implement the same pattern used for interface management:
- Detect route changes in `Update` function
- Call appropriate API endpoints to add/remove routes
- Exclude routes from PATCH request body

## Related Issues

This bug is similar to previous interface handling issues that were fixed by implementing the attach/detach pattern instead of relying on PATCH updates.
