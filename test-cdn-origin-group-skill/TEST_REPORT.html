Technical Details

Root Cause

The s3_credentials_version field had a json:"-" tag which caused apijson.UnmarshalComputed to reset it to null. Other resources with similar version fields (e.g., cloud_inference_secret.data_wo_version) use only the tfsdk tag without any json tag.

Solution

Removed the json:"-" tag from the field definition:

// Before (buggy):
S3CredentialsVersion types.Int64 `tfsdk:"s3_credentials_version" json:"-"`

// After (fixed):
S3CredentialsVersion types.Int64 `tfsdk:"s3_credentials_version"`

This matches the pattern used in other resources with write-only fields and version triggers.
