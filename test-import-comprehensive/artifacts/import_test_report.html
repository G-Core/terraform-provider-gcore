Bugs Found & Fixed

BUG 1: Tags not imported FIXED

Location: internal/services/cloud_instance/resource.go:1589-1613

Problem: Tags were not being read from API response during import.

Solution: Added extraction logic in ImportState function that:

  • Parses tags from API response format: [{"key": "k", "value": "v", "read_only": bool}, ...]
  • Filters out system-generated tags (read_only=true) like image_id, os_distro, etc.
  • Populates the data.Tags map with user-defined tags only
BUG 2: servergroup_id not imported FIXED

Location: internal/services/cloud_instance/resource.go:1616-1656

Problem: Instance API doesn't return servergroup_id, so it was missing after import.

Solution: Added placement group lookup in ImportState function that:

  • Lists all placement groups in the project/region
  • Gets full details of each placement group (List API returns empty instances array)
  • Checks if the imported instance belongs to any placement group
  • Populates data.ServergroupID if a match is found
