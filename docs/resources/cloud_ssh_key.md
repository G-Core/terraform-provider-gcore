# gcore_cloud_ssh_key (Resource)

SSH key pairs provide secure authentication to cloud instances, supporting both generated and imported public keys.

## Example Usage

### Upload an existing SSH key

Add an SSH public key that can be used when creating instances.

```terraform
# Upload an existing SSH public key
resource "gcore_cloud_ssh_key" "deployer" {
  project_id = 1
  name       = "deployer-key"
  public_key = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIJe8rDJP... deployer@workstation"
}
```

~> **Tip:** Generate a key pair locally with `ssh-keygen -t ed25519`, then paste the contents of the `.pub` file into `public_key`.

## Schema

### Required

- `name` (String) SSH key name
- `public_key` (String) The public part of an SSH key is the shareable portion of an SSH key pair. It can be safely sent to servers or services to grant access. It does not contain sensitive information. You must provide your own public key (usually found in a file like `id_ed25519.pub` or `id_rsa.pub`). Generate your SSH keypair locally using `ssh-keygen` before providing it here.

### Optional

- `project_id` (Number) Project ID
- `shared_in_project` (Boolean) SSH key is shared with all users in the project

### Read-Only

- `created_at` (String) SSH key creation time
- `fingerprint` (String) Fingerprint
- `id` (String) SSH key ID
- `state` (String) SSH key state
Available values: "ACTIVE", "DELETING".
