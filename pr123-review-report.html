Comment #2 — Use typed SDK params instead of raw JSON RESOLVED
Original Comment (inline, line 199-200)
The SDK's CDNResourceRuleUpdateParams already has an Active param.Opt[bool] field (confirmed at gcore-go@v0.33.0/cdn/cdnresourcerule.go:4632). Using a raw JSON literal here bypasses SDK serialization and is the only instance of this pattern across all 57 WithRequestBody call sites in the codebase — every other one uses a serialized struct.

This would be more robust as:
_, err := r.client.CDN.CDNResources.Rules.Update(
    ctx,
    data.ID.ValueInt64(),
    cdn.CDNResourceRuleUpdateParams{
        ResourceID: data.ResourceID.ValueInt64(),
        Active:     param.NewOpt(false),
    },
    option.WithMiddleware(logging.Middleware(ctx)),
)
Type-safe, consistent with the rest of the provider, and won't silently conflict if the params struct evolves. You'd need to add "github.com/G-Core/gcore-go/packages/param" to the imports. 🤖
