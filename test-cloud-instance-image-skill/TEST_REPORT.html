Known Issues / Expected Behavior

  • url forces replacement when specified after import: This is expected. The url field is write-only (used during creation only, not returned by API). If you import an image and then add url to your config, Terraform will show "forces replacement" because url triggers recreation. To avoid this, omit url from config after import.
  • Computed fields show "(known after apply)" during update: Fields like description, display_order, gpu_driver_*, status, task_id, updated_at show as "(known after apply)" during updates. This is acceptable behavior for computed fields - they are refreshed after each operation.
  • Image status transitions: After create, status is "queued" as image downloads. Status becomes "active" once download completes. This is normal async behavior.
  • Write-only fields (no_refresh): Fields with no_refresh tag (url, os_distro, os_version, cow_format, tags) are not returned by API GET. After import, these appear as additions. This is expected for write-only fields.
