# Comprehensive QA Checklist for Terraform Resources

## Critical Issues to Test (Based on Production Findings)

This checklist is compiled from real issues found in production Terraform providers, including insights from QA testing of autogenerated providers.

## 1. Drift Detection Issues ⚠️ MOST CRITICAL

**Problem**: Resource shows perpetual drift - `terraform plan` always detects changes after clean apply.

### Checklist:
- [ ] **Test drift immediately**: `terraform apply && terraform plan -detailed-exitcode`
- [ ] **Verify UnmarshalComputed usage** in ALL methods:
  - [ ] Read method uses `apijson.UnmarshalComputed`
  - [ ] ImportState uses `apijson.UnmarshalComputed`
  - [ ] Update method uses `UnmarshalComputed` for response
- [ ] **Check computed_optional fields**:
  - [ ] Model has `json:"field_name,computed_optional"` tag
  - [ ] Schema has `Computed: true, Optional: true`
- [ ] **Test with minimal config** (let API compute defaults)
- [ ] **Test with full config** (provide all values)

### Common Drift-Causing Fields:
```
- http_method (defaults to GET)
- max_retries_down (defaults to 3)
- timeout values
- protocol ports
- any field with server-side defaults
```

## 2. Update vs Recreate Issues

**Problem**: Updates cause resource recreation instead of in-place modification.

### Checklist:
- [ ] **Verify Update method exists** for the resource
- [ ] **Check ForceNew/RequiresReplace** only on immutable fields
- [ ] **Test each updatable field**:
  ```bash
  initial_id=$(terraform show -json | jq -r '.values.root_module.resources[0].values.id')
  terraform apply -var="field_updated=true"
  new_id=$(terraform show -json | jq -r '.values.root_module.resources[0].values.id')
  [ "$initial_id" = "$new_id" ] || echo "FAIL: Resource recreated!"
  ```
- [ ] **Verify PATCH operations** in mitmproxy logs
- [ ] **Test complex nested updates** (arrays, objects)

## 3. Async vs Sync Operations

**Problem**: Incorrect handling of task-based operations.

### Checklist:
- [ ] **Identify operation type**:
  - [ ] Check for task IDs in API response
  - [ ] Look for `WaitTaskAndReturn` in old provider
- [ ] **Verify polling implementation**:
  - [ ] Async uses `*AndPoll` methods
  - [ ] Sync uses direct methods
- [ ] **Test task failure scenarios**
- [ ] **Verify timeout handling**

## 4. Array and List Handling

**Problem**: Incorrect handling of empty arrays vs null.

### Checklist:
- [ ] **Test empty array**: `interfaces = []`
- [ ] **Test null/omitted**: Don't specify field
- [ ] **Test array updates**:
  - [ ] Add items
  - [ ] Remove items
  - [ ] Clear all (should send `[]`)
- [ ] **Verify API payload** in mitmproxy:
  ```json
  // Clearing should send empty array
  {"routes": []}
  // Not null or omitted
  ```

## 5. Nested Object Updates

**Problem**: Nested objects not updating correctly.

### Checklist:
- [ ] **Test partial updates** of nested objects
- [ ] **Verify no unintended side effects**
- [ ] **Check computed nested fields**
- [ ] **Test removing nested objects**

## 6. Import State Functionality

**Problem**: Import doesn't work or causes drift.

### Checklist:
- [ ] **Test import**: `terraform import resource.name <id>`
- [ ] **Verify no drift after import**
- [ ] **Check all fields populated correctly**
- [ ] **Test with complex resources**

## 7. Validation and Error Handling

**Problem**: Poor error messages or missing validation.

### Checklist:
- [ ] **Test invalid values** for each field
- [ ] **Verify clear error messages**
- [ ] **Test API error scenarios**:
  - [ ] Quota exceeded
  - [ ] Invalid parameters
  - [ ] Network failures
- [ ] **Check timeout handling**

## 8. Computed Field Management

**Problem**: Computed fields not handled correctly.

### Checklist:
- [ ] **Identify all computed fields** from API docs
- [ ] **Verify read-only enforcement**
- [ ] **Test computed_optional pattern**:
  - [ ] User provides value: use it
  - [ ] User omits: use API default
  - [ ] No drift in either case

## 9. State Consistency

**Problem**: State doesn't match actual infrastructure.

### Checklist:
- [ ] **Compare state to API GET**:
  ```bash
  terraform show -json | jq '.values.root_module.resources[0].values'
  # Compare with actual API response
  ```
- [ ] **Test after external changes**
- [ ] **Verify refresh updates state**
- [ ] **Check sensitive field handling**

## 10. Resource Dependencies

**Problem**: Dependencies not handled correctly.

### Checklist:
- [ ] **Test creation order**
- [ ] **Test deletion order** (reverse)
- [ ] **Verify cascade deletes**
- [ ] **Test circular dependencies** (should fail gracefully)

## 11. Special Operations

**Problem**: Special endpoints not used correctly.

### Router-specific:
- [ ] Interface attach/detach uses special endpoints
- [ ] Routes update uses PATCH with full array

### Load Balancer-specific:
- [ ] Pool member operations
- [ ] Health monitor updates
- [ ] Certificate management

### Network-specific:
- [ ] Subnet allocation
- [ ] Security group rules
- [ ] Floating IP assignment

## 12. Performance Testing

### Checklist:
- [ ] **Test with many resources** (100+)
- [ ] **Measure operation times**
- [ ] **Check for unnecessary API calls**
- [ ] **Verify pagination handling**

## 13. Edge Cases

### Checklist:
- [ ] **Maximum field lengths**
- [ ] **Special characters in strings**
- [ ] **Unicode handling**
- [ ] **Very large arrays/lists**
- [ ] **Deeply nested structures**

## Testing Priority by Resource Type

### High Priority (Complex, Stateful):
1. Load Balancers (pools, listeners, members)
2. Routers (interfaces, routes)
3. Instances (with volumes, networks)
4. Kubernetes Clusters

### Medium Priority:
1. Networks and Subnets
2. Security Groups
3. Volumes
4. Floating IPs

### Lower Priority (Simple, Stateless):
1. SSH Keys
2. Projects
3. DNS Records
4. Object Storage

## Automation Tips

### Generate Test for All Resources:
```python
# Use generate_test_config.py for each resource type
for resource in ['router', 'load_balancer', 'instance', 'volume']:
    generate_test_config.py $resource
```

### Batch Testing:
```bash
# Test all resources systematically
for dir in tests/*/; do
    resource=$(basename $dir)
    ./scripts/run_tests.sh $resource
done
```

### Continuous Validation:
- Run drift tests nightly
- Test after provider updates
- Monitor for API changes

## Common Fix Patterns

### Pattern 1: Fix Drift
```go
// Change all occurrences:
- apijson.Unmarshal(bytes, &data)
+ apijson.UnmarshalComputed(bytes, &data)
```

### Pattern 2: Fix Computed Optional
```go
// Model:
- Field types.String `json:"field,optional"`
+ Field types.String `json:"field,computed_optional"`

// Schema:
"field": schema.StringAttribute{
+   Computed: true,
    Optional: true,
}
```

### Pattern 3: Fix Array Updates
```go
// Send empty array to clear, not null
if len(data.Routes.Elements()) == 0 {
    updateData["routes"] = []interface{}{}
}
```
